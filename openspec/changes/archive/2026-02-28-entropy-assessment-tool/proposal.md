## Why

用户需要一种结构化的自我评估方式来衡量自身的"熵值"状态。通过标准化量表和明确的评分规则，帮助用户了解自己目前处于低熵、中熵还是高熵状态，从而获得有针对性的参考信息。

## What Changes

- 新增一个包含 32 条题目的评估量表页面，每题提供 5 个选项供用户选择
- 新增每题各选项的得分表，不同题目、不同选项对应不同分值
- 新增总熵值计算逻辑：将用户所有题目的得分累加，得出总分（32～160 分）
- 新增评估结果页面，展示用户的总熵值及所处熵段（低熵/中熵/高熵）

## Capabilities

### New Capabilities

- `assessment-scale`: 32 条题目的评估量表，每题提供"完全不符 / 不太符合 / 一般 / 比较符合 / 完全符合"五个选项，用户逐题作答
- `scoring-table`: 各条目不同选项对应的得分数据表，是计算总熵值的数据基础
- `entropy-calculation`: 总熵值计算逻辑，将用户 32 道题的选项得分累加，得出 32～160 的总分
- `result-display`: 结果展示页面，显示用户的总熵值分数，并根据分段规则（低熵 32～64 / 中熵 65～127 / 高熵 128～160）展示对应的熵段状态

### Modified Capabilities

（无现有规格变更）

## Impact

- 纯前端或全栈新项目，无现有代码依赖
- 核心数据：32 条题目文本 + 各题各选项得分表（需由产品方提供或确认）
- 评分规则为静态数据，无需数据库，可内置于代码或配置文件中
- 结果分段规则固定：32～64 低熵，65～127 中熵，128～160 高熵
